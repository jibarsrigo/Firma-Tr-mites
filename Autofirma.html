AutoFirma – Diagnóstico exhaustivo y recuperación avanzada (CAU)

Documento técnico interno para personal CAU. Contiene todo el conocimiento operativo sobre firma electrónica con FIRE, AutoFirma cliente, AutoFirma servidor y firma móvil.

NO es documentación para ciudadanos.

1. Marco general y principio rector

Cuando un ciudadano firma con certificado local (FNMT, DNIe u otros), el proveedor funcional del flujo es siempre FIRE.

  • AutoFirma cliente NO es proveedor.
  • AutoFirma cliente es una herramienta local del entorno del ciudadano.
  • AutoFirma servidor o componentes intermedios se analizan solo si aparece SAF_27.

Regla CAU: no escalar como proveedor sin descartar entorno local.

2. Regla universal de diagnóstico

OBLIGATORIO: exigir siempre prueba de firma local con PDF.

  1. Abrir AutoFirma manualmente.
  2. Seleccionar “Firmar archivo / Firmar fichero”.
  3. Firmar un PDF local.
Resultado Diagnóstico Acción
No abre AutoFirma Instalación / permisos No escalar
No aparece certificado Certificado / DNIe No escalar
No firma PDF AutoFirma / entorno No escalar
Firma PDF OK Invocación web / navegador Revisar entorno

3. Patrón maestro: firma local OK / firma web KO

  • AutoFirma firma PDF local correctamente.
  • La firma del trámite web no se completa.
  • Aparece TR_SGI, pero no TR_SGO.

Interpretación: problema de comunicación navegador ↔ AutoFirma, no del certificado ni del proveedor.

  • Cerrar navegador.
  • Probar otro navegador.
  • Usar “Herramientas → Restaurar instalación”.
  • Probar sin VPN / red corporativa.

4. Relación SistraHelp ↔ diagnóstico real

Patrón de trazas Interpretación CAU

5. Diagnóstico por sistema operativo

Windows

  • AutoFirma no se abre o no se invoca.
  • Persistencia de .afirma tras desinstalar.
  • Firewall bloquea AutoFirma aunque antivirus no.
  • Chrome / Edge más restrictivos que Firefox.
  • Registro incorrecto del protocolo afirma://.

macOS

  • AutoFirma abre.
  • Firma local OK.
  • Firma web KO.
  • Gatekeeper y llavero Sistema vs Inicio de sesión.
  • Apple Silicon + macOS recientes: puertos dinámicos.

Linux

  • Firefox Snap / Flatpak aislado.
  • PKCS#11 obligatorio para DNIe.
  • Certificado visible en navegador pero no en AutoFirma.

Android

  • AutoFirma App firma PDF local.
  • Solo firma trámites compatibles.
  • DNIe por NFC inestable.

iPhone / iPad

  • AutoFirma App firma PDF local.
  • Safari debe usar vista móvil.
  • No todos los trámites son compatibles.

6. Diagnóstico por tipo de certificado

FNMT / certificado software

  • Certificado no caducado.
  • Clave privada presente.
  • Archivo original .p12 / .pfx.

DNIe

  • DNIe en vigor.
  • PIN no bloqueado.
  • Lector reconocido.
  • Drivers / PKCS#11 instalados.

7. Keystore interno de AutoFirma

AutoFirma utiliza un keystore SSL interno con AutoFirma ROOT y certificado de servidor 127.0.0.1. No contiene certificados personales.

  • Errores SSL.
  • SAF_36 / SAF_37.
  • Firma local OK pero web KO.

Acción: Restaurar instalación o reinstalación limpia.

8. Limpieza completa de AutoFirma (Windows)

SOLO personal técnico.

  1. Desinstalar AutoFirma.
  2. Reiniciar.
  3. Eliminar:
    • C:\Program Files\AutoFirma
    • C:\Users\usuario\.afirma
  4. Eliminar claves de registro (si procede).
  5. Reiniciar.
  6. Instalar como administrador.

9. Criterios de escalado

NO escalar si:

  • Un solo NIF.
  • Firma local KO.
  • Falla solo en un equipo o red.
  • No aparece SAF_27.

SÍ escalar si:

  • Varios NIF afectados.
  • Misma franja horaria.
  • SAF_27 masivo.

10. Regla de oro CAIB

Si aparece 127.0.0.1, wss:// o AutoFirma en el error, NO es un problema de la sede CAIB ni de @firma/FIRE.
